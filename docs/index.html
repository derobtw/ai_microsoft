Workflow

From file upload to grounded answer.

The project keeps the RAG pipeline visible: ingestion, retrieval, generation, and sources.

01

Upload documents

The local Flask app accepts PDF, DOCX, and TXT files through a minimal web interface.

02

Index locally

Text is extracted, chunked, embedded, and stored in a session SQLite database.

03

Retrieve context

Questions are matched against document chunks using similarity and keyword-aware filtering.

04

Answer with sources

The model answers only from retrieved context and shows where the answer came from.

Project overview

A complete local RAG demo, not just a chatbot.

The final project demonstrates model setup, embeddings, SQLite storage, retrieval filtering, PDF/DOCX support, and a browser-based user experience.

Foundry Local

Runs the chat and embedding models on the user machine during the local demo.

SQLite sessions

Creates a separate local database for each browser session and uploaded document set.

Document support

Indexes TXT, PDF, and DOCX files with source labels and chunk locations.

Grounding guardrails

Blocks weak matches so the assistant is less likely to answer from memory.
